# ID: NP2-03

## Title

Data versioning UX for all “data-backed” tools

## Context

Salary tool already displays data freshness via same-origin feed. Extend the same contract to any tool that depends on versioned data to improve trust and reduce support load.

## Steps

1. Define a shared “data version badge” component.
2. Ensure each relevant tool has:
   - local embedded data version
   - same-origin feed version (optional)
3. Add tests that prevent stale/unknown version output.

## Acceptance Criteria

- Users can see “last updated” and data source for relevant tools.

## Dependencies

- None.

## Files to modify

- `lib/finance-data-version.ts`
- `components/features/finance/*`
- `components/features/salary/SalaryPage.tsx`
