# ID: NP1-07

## Title

Tool-tier enforcement: ensure Online-Required is truly network-only end-to-end

## Context

There is a tier system (`lib/tools-registry.policy.ts`) and SW special-cases `/pro`. Expand this to ensure Online-Required routes:

- are not cached by SW,
- show correct offline UX,
- are excluded from offline guarantees.

## Steps

1. Define Online-Required route list/prefixes (beyond `/pro` if applicable).
2. Enforce in SW fetch handler and caching lists.
3. Add e2e offline tests for Online-Required routes behavior.

## Acceptance Criteria

- Online-Required routes never load stale cached content.
- Offline mode shows a clear message for Online-Required routes.

## Dependencies

- None.

## Files to modify

- `lib/tools-registry.policy.ts`
- `public/sw.js`
- `tests/e2e/offline.spec.ts`
