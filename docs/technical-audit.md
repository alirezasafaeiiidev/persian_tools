### 2. دستههای اصلی ممیزی

#### کیفیت کد

- تشخیص کد مرده (`tsc --noEmit`، ts-morph)
- اجرای strict TypeScript (اجتناب از `any`)
- کاهش وابستگیهای runtime
- هدف پوشش تست ≥ 85% (Vitest/Playwright)

#### انطباق UI/UX

- پشتیبانی کامل RTL (`dir="rtl"`)
- سازگاری تایپوگرافی فارسی (مثل IRANSansX)
- دسترسپذیری WCAG 2.1 AA:
  - نقشهای ARIA
  - ویژگیهای alt
  - ناوبری با کیبورد
- ریسپانسیو mobile-first (≥ 95%)

#### SEO و عملکرد

- تکمیل meta/OG
- دادههای ساختیافته (schema.org SoftwareApplication)
- اهداف Lighthouse:
  - امتیاز موبایل ≥ 95
  - زمان بارگذاری < 1.5s روی 3G
- بهینهسازی Core Web Vitals

#### تمایز رقابتی

- قابلیتهای مفقود

#### ابزارهای توسعه و DevOps
