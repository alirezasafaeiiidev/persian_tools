## اصل کلیدی: Local‑First
پروژه باید به‌صورت پیش‌فرض **کاملاً قابل استفاده بدون اینترنت جهانی** باشد. هر وابستگی خارجی باید:
- **صراحتاً دسته‌بندی شود** (Hybrid یا Online‑Required)
- **فقط از طریق دامنه خود پروژه** مصرف شود (کلاینت هیچ fetch خارجی انجام ندهد)
- **دارای تجربه کاربری مشخص در حالت قطعی اینترنت/تحریم** باشد

## تعریف سریع “۱۰/۱۰” (Definition of Done سطح محصول)
برای اینکه پروژه از همه لحاظ ۱۰/۱۰ محسوب شود، این معیارها باید برقرار باشند:

### Local‑First
- هیچ درخواست runtime (fetch/beacon/script/import) به دامنه‌های خارجی انجام نشود.
- سرویس‌ورکر و کش باید تجربه آفلاین پایدار بدهند.
- ابزارهای Online‑Required باید به‌صورت سخت‌گیرانه Network‑Only باشند و پیام مناسب ارائه کنند.

### UX/UI
- ورودی‌های عددی و تاریخ استاندارد، موبایل‌پسند، و خطاپذیر باشند.
- فرم‌های سنگین به مراحل قابل فهم شکسته شوند.
- حداقل WCAG AA برای دسترسی‌پذیری رعایت شود.

### SEO
- URLها پایدار و کوتاه؛ canonical و metadata یکتا.
- JSON‑LD معتبر و یکپارچه؛ sitemap تمیز (بدون صفحات Coming‑Soon ایندکس‌شده).

### Performance
- JS اولیه حداقلی؛ ابزارهای سنگین lazy‑load.
- فونت‌ها WOFF2 و preload درست.
- تعاملات (INP) روی موبایل متوسط روان باشد.

### Correctness
- برای ابزارهای حیاتی (Loan/Salary/Interest) تست‌های طلایی و وکتورهای واقعی وجود داشته باشد.
- فرضیات محاسبه و منبع داده/قانون شفاف نمایش داده شود.
